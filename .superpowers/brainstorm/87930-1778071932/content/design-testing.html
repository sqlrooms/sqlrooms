Testing & Verification

How to verify the implementation works correctly

Testing Strategy

1. Unit Tests (Component Level)

  • ColumnSelector - renders columns, filters by types, calls onChange with selected value
  • AggregationSelector - renders 4 options, calls onChange with selected function
  • FieldSelector - renders label with required indicator, lays out children
  • MultiFieldSelector - manages array add/update/remove, emits full updated array
  • LineChartSettings - composes primitives correctly, updates full settings object

2. Integration Tests

  • ChartSettingsPanel routes to LineChartSettings for line-chart
  • ChartSettingsPanel falls back to DynamicChartSettings for other charts
  • Settings updates propagate correctly to chart spec generation
  • Temporal granularity shows/hides based on xInterval
  • Aggregation shows/hides in MultiFieldSelector based on prop

3. Manual Testing

  • Run example app with line chart
  • Select X field, verify temporal selector appears for date/timestamp columns
  • Add multiple Y fields, verify aggregation selector appears when xInterval is set
  • Remove Y fields, verify array updates correctly
  • Change aggregation, verify chart updates
  • Test with other chart types to ensure no regressions

Edge Cases to Handle

  • Empty columns list - disable selectors, show message
  • No matching columns for types - show "No columns available" message
  • Remove last Y field - prevent removal, require at least one
  • Selected column removed from schema - clear invalid selection, show warning
  • Change X field from temporal to non-temporal - clear xInterval automatically
  • Duplicate field selection - prevent adding same column twice in MultiFieldSelector
  • Invalid settings object - validate with Zod schema, show error

Verification Checklist

✅ All new components render without errors
✅ LineChartSettings composes components correctly
✅ ChartSettingsPanel routes to correct component
✅ Settings updates trigger chart re-render
✅ Generated Mosaic spec matches expected structure
✅ Temporal granularity conditional rendering works
✅ Aggregation conditional rendering works
✅ Multi-field add/update/remove works
✅ Type filtering in column selectors works
✅ Existing chart types still work with DynamicChartSettings
✅ No TypeScript errors
✅ No console warnings/errors

Performance Considerations

  • Memoization - use React.memo on primitives to prevent unnecessary re-renders
  • useCallback - wrap onChange handlers in LineChartSettings
  • useMemo - memoize filtered columns lists
  • Avoid inline objects - don't create new objects in render (causes re-renders)
  • Key stability - use stable keys in MultiFieldSelector rows (field name, not index)
