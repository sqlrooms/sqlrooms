State Management: onChange Handling

How should compound components manage state updates?

A

Controlled at Chart Level

Each chart settings component receives values and onChange, manages the full state object internally.

// LineChartSettings.tsx
<FieldSelector
value={values.x}
onChange={(x) => onChange({...values, x})}
/>

<MultiFieldSelector
value={values.yFields}
onChange={(yFields) => onChange({...values, yFields})}
/>

Pros

  • Simple API
  • Chart component owns logic
  • Easy validation

Cons

  • Repetitive onChange wiring
  • Hard to share state between fields
B

Context Provider

Compound component pattern with Context - parent provides state, children consume it.

// Usage
<LineChartSettings
values={values}
onChange={onChange}
>
<LineChartSettings.XAxis />
<LineChartSettings.YAxis />
</LineChartSettings>

// Subcomponents read/write via context
// const {values, updateField} = useChartSettings()

Pros

  • Clean composition
  • Shared state access
  • No prop drilling

Cons

  • More boilerplate
  • Context overhead
C

Hybrid: Explicit Props + Helpers

Chart component uses explicit props but provides helper functions to reduce boilerplate.

// LineChartSettings.tsx
const updateField = useCallback(
(key, val) => onChange({...values, [key]: val}),
[values, onChange]
);

<FieldSelector
value={values.x}
onChange={(x) => updateField('x', x)}
/>

<MultiFieldSelector
value={values.yFields}
onChange={(yFields) => updateField('yFields', yFields)}
/>

Pros

  • Explicit data flow
  • Less boilerplate than A
  • No context complexity

Cons

  • Still some wiring needed
  • Helper per component
D

Form Library (React Hook Form / Formik)

Use a form library to manage state and validation.

// LineChartSettings.tsx
const form = useForm({
defaultValues: values,
onSubmit: onChange
});

<FieldSelector
{...form.register('x')}
/>

<MultiFieldSelector
{...form.register('yFields')}
/>

Pros

  • Battle-tested
  • Built-in validation
  • Performance optimized

Cons

  • External dependency
  • Learning curve
  • May be overkill

Key Questions

  • Do you need cross-field validation? (e.g., "if x is temporal, require xInterval")
  • Should state updates be immediate or batched?
  • Do you want the compound components (FieldSelector, MultiFieldSelector) to be reusable outside chart settings?
