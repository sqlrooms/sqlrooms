FieldSelector Component Pattern

How should the single FieldSelector work?

We confirmed MultiFieldSelector will be self-contained (Option A). Now let's clarify FieldSelector for single fields like X-axis.

A

Simple Wrapper (Just Layout)

FieldSelector is just a layout container with label. Child components handle their own onChange.

// Usage
<FieldSelector label="X Axis" required>
<ColumnSelector
columns={columns}
types={['DATE', 'TIMESTAMP']}
value={values.x}
onChange={(x) => updateField('x', x)}
/>
{values.x && <TemporalGranularitySelector
value={values.xInterval}
onChange={(interval) => updateField('xInterval', interval)}
/>}
</FieldSelector>
// Implementation
function FieldSelector({label, required, children}) {
return (
<div className="space-y-1">
<label>{label} {required && '*'}</label>
<div className="grid gap-2">{children}</div>
</div>
);
}

Pros

  • Simple and composable
  • Parent controls all state
  • Maximum flexibility
  • Clear data flow

Cons

  • Parent handles each onChange
  • More wiring in chart settings
B

Smart Container (Manages Multiple Fields)

FieldSelector coordinates multiple related fields (e.g., x + xInterval) and returns them together.

// Usage
<FieldSelector
label="X Axis"
required
columns={columns}
types={['DATE', 'TIMESTAMP']}
value={{x: values.x, xInterval: values.xInterval}}
onChange={(fields) => onChange({...values, ...fields})}
showTemporal
/>
// Implementation (simplified)
function FieldSelector({value, onChange, showTemporal, ...}) {
return (
<>
<ColumnSelector
value={value.x}
onChange={(x) => onChange({...value, x})}
/>
{showTemporal && value.x && <TemporalGranularitySelector
value={value.xInterval}
onChange={(xInterval) => onChange({...value, xInterval})}
/>}
</>
);
}

Pros

  • Encapsulates related fields
  • Less wiring in parent
  • Consistent with MultiFieldSelector

Cons

  • Less flexible
  • Hard-coded field relationships
  • Not reusable for simple cases
