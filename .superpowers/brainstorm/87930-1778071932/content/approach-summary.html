Confirmed Decisions

  • Component Location: Chart settings co-located with chart types (chart-types/line-chart/LineChartSettings.tsx)
  • Component Reuse: Build new components alongside existing ones, gradual migration
  • Migration Scope: Start with line-chart only, keep DynamicChartSettings as fallback
  • MultiFieldSelector API: Self-contained, returns full updated array
  • FieldSelector Pattern: Simple layout wrapper, children handle their own onChange

Implementation Approach

Phase 1: New Primitives

Create new compound components in chart-builders/:

  • FieldSelector - Simple wrapper with label + grid layout
  • ColumnSelector - Reusable column picker (wraps existing FieldSelectorInput)
  • AggregationSelector - Dropdown for SUM/AVG/MIN/MAX
  • MultiFieldSelector - Enhanced version with compound pattern

These work alongside existing components.

Phase 2: LineChartSettings

Create explicit settings component:

  • Location: chart-types/line-chart/LineChartSettings.tsx
  • Props: columns, values, onChange
  • Composed with new primitives (FieldSelector, ColumnSelector, etc.)
  • Handles line-chart-specific logic (temporal granularity, aggregation)

Phase 3: Integration

Wire up in ChartSettingsPanel:

  • Check if chart type has explicit settings component
  • If yes: render <LineChartSettings />
  • If no: fallback to <DynamicChartSettings />
  • Other chart types continue using DynamicChartSettings

// chart-types/line-chart/LineChartSettings.tsx

export function LineChartSettings({
columns: ChartBuilderColumn[],
values: LineChartSettings,
onChange: (values: LineChartSettings) => void
}) {
const updateField = useCallback(
(key, value) => onChange({...values, [key]: value}),
[values, onChange]
);

return (
<div className="space-y-4">
{/* X Axis */}
<FieldSelector label="X Axis" required>
<ColumnSelector
columns={columns}
types={QUANTITATIVE_COLUMN_TYPES}
value={values.x}
onChange={(x) => updateField('x', x)}
/>
{values.x && <TemporalGranularitySelector
value={values.xInterval}
onChange={(i) => updateField('xInterval', i)}
/>}
</FieldSelector>

{/* Y Axes */}
<MultiFieldSelector
label="Y Axes"
columns={columns}
types={NUMERIC_COLUMN_TYPES}
value={values.yFields}
onChange={(yFields) => updateField('yFields', yFields)}
showAggregation={Boolean(values.xInterval)}
/>
</div>
);
}

Benefits of This Approach

  • Gradual migration - No big bang rewrite, test with one chart first
  • No breaking changes - Existing charts continue working with DynamicChartSettings
  • Full flexibility - Each chart can customize its settings UI completely
  • Reusable primitives - New components can be used in other contexts too
  • Clear ownership - Chart settings live next to chart logic
