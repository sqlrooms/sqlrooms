2. Component Reuse Strategy

You already have:

  • FieldSelectorInput (column picker with type filtering)
  • TemporalGranularitySelector
  • MultiFieldSelector (with aggregation support)
  • SingleFieldSelector
A

Refactor Existing Components

Rename/refactor existing components to match new pattern:

  • FieldSelectorInput → ColumnSelector
  • MultiFieldSelector → keep, enhance API
  • Add new FieldSelector (simple wrapper)
  • Add new AggregationSelector (extract from MultiFieldSelector)
B

Build New, Keep Old (Gradual Migration)

Create new components alongside existing ones:

  • Keep FieldSelectorInput, SingleFieldSelector, etc.
  • Create new FieldSelector, ColumnSelector, etc.
  • New chart types use new components
  • Migrate old charts gradually

3. Migration Scope

A

Migrate All Chart Types

Convert all existing charts (line-chart, histogram, bubble-chart, etc.) to new pattern and remove DynamicChartSettings.

B

Start with Line Chart Only

Prove the pattern works with line-chart first, keep DynamicChartSettings as fallback for others, migrate incrementally.

My Recommendation

  • Structure: Option A - co-locate settings with chart types
  • Reuse: Option A - refactor existing components (they're already 90% there)
  • Scope: Option B - start with line-chart, prove it works, then migrate
