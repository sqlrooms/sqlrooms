Data Flow & State Management

How data flows through the component hierarchy

State Ownership

  • Chart Settings Component - receives values and onChange from parent, owns the full settings object
  • MultiFieldSelector - receives value (array) and onChange, manages array mutations internally
  • FieldSelector - stateless layout wrapper, no state management
  • Primitives - controlled components, receive value and onChange

Update Flow Example

// User changes Y field column
ColumnSelector
.onChange('revenue') // primitive fires
↓
MultiFieldSelector
.handleUpdate(index, {field: 'revenue'}) // compound updates array
↓
MultiFieldSelector
.onChange([{field: 'revenue', aggregate: 'sum'}]) // emits full array
↓
LineChartSettings
.updateField('yFields', [...]) // updates full settings
↓
ChartSettingsPanel
.onChange({x: ..., yFields: [...], xInterval: ...}) // emits to parent

Key Principles

  • Unidirectional flow - data flows down, events flow up
  • Full object updates - always emit complete updated object/array, never partial updates
  • Immutable updates - spread operators, no mutations
  • Single source of truth - chart settings component owns the full settings object
  • No prop drilling - but also no context magic, explicit props all the way
