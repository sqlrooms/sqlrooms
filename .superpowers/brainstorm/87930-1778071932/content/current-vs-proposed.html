Chart Settings Architecture

Current DynamicChartSettings vs. Proposed Explicit Components

Current: DynamicChartSettings
// Single component handles all chart types
<DynamicChartSettings
chartTypeDefinition={lineChartChartType}
columns={columns}
values={values}
onChange={onChange}
/>

// Internally decides what to render
// - SingleFieldSelector for x
// - MultiFieldSelector for yFields
// - FieldSelectorInput for others

Issues:
• Limited flexibility
• Hard to customize per chart type
• Tight coupling to field definitions
Proposed: Explicit Components
// Each chart has dedicated settings component
<LineChartSettings
columns={columns}
values={values}
onChange={onChange}
/>

// Composed with reusable primitives:

<FieldSelector label="X Axis" ...>
<ColumnSelector />
<TemporalGranularitySelector />
</FieldSelector>

<MultiFieldSelector label="Y Axes" ...>
{fields.map(() => (
<ColumnSelector />
<AggregationSelector />
<DeleteButton />
))}
</MultiFieldSelector>

Benefits:
• Full flexibility per chart
• Reusable compound components
• Clear component boundaries
